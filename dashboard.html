💡 Note technique — Intégration SQLite

-- En production, le feedback est sauvegardé dans SQLite via fraudx/db.py :
-- Table: feedback (id, transaction_id, is_fraud, analyst, comment, created_at)
-- La table predictions (id, transaction_id, fraud_score, prediction, risk_level, raw_input)
-- Permet de calculer les métriques réelles (F1, Recall) vs feedbacks analystes.
-- Le réentraînement automatique (retrain.py) utilise ces feedbacks pour améliorer le modèle.
